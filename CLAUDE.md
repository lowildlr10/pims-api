## Architecture

All API routes are versioned under `/api/v1/` and defined in `routes/v1/api.php`. Controllers live in `app/Http/Controllers/V1/` organized into four domain modules:

| Module | Path | Scope |
|--------|------|-------|
| Account | `V1/Account/` | Auth, users, roles, departments, sections |
| Procurement | `V1/Procurement/` | Purchase requests â†’ quotations â†’ orders â†’ vouchers |
| Inventory | `V1/Inventory/` | Supplies and issuances |
| Library | `V1/Library/` | Reference/lookup data (suppliers, units, classifications) |

### Request â†’ Response flow

```
Request
  â†’ Controller  (HTTP handling, validation, thin)
  â†’ Service     (business logic, filter parsing, logging)
  â†’ Repository  (Eloquent queries, pagination, PDF generation)
  â†’ Model       (UUIDs as PKs, Enum casts, status_timestamps JSON column)
  â†’ Resource    (JSON transformer, always use whenLoaded() for relationships)
```

Repository contracts (interfaces) live in `app/Interfaces/` (some older ones in `app/Contracts/`). Bindings are registered in `app/Providers/AppServiceProvider.php`.

| Element | Convention | Example |
|---------|------------|---------|
| Models | PascalCase singular | `PurchaseRequest` |

| Methods | camelCase | `parseFilters()`, `storeUpdate()` |
| Relationships | snake_case | `funding_source()` |

### Key conventions

- **Models** use `HasUuids` for primary keys. Status fields are cast to PHP 8 backed Enums (all in `app/Enums/`). Enums expose a `label()` method for human-readable values. The `status_timestamps` JSON column is managed via `StatusTimestampsHelper::generate('status_key', $current)`.
- **Services** always have a `parseFilters()` method and a `logError()` method. Boolean query params must be coerced with `filter_var(..., FILTER_VALIDATE_BOOLEAN)`.
- **Repositories** handle both paginated and non-paginated results using the `paginated`/`show_all` filter flags. Some use `storeUpdate(array $data, ?Model $model = null)` for create-or-update logic instead of separate `create()`/`update()`.
- **Controllers** wrap mutation calls in `try/catch (\Throwable $th)` and delegate to `$service->logError()` on failure; return `422` JSON on error.
- **Resources** expose both raw and formatted fields (e.g., `status` + `status_formatted`, `pr_date` + `pr_date_formatted`).
- **Logging** uses `LogRepository::create([..., 'log_module' => 'pr', ...])`. Notifications use `NotificationRepository::notify()`.
- **Scribe annotations** (`@group`, `@queryParam`, `@bodyParam`, `@response`) are required on all controller methods. Generated docs are served at `/docs`.
- Imports are alphabetically ordered; type hints and return types are required everywhere.

### Permission system

Routes are gated with `middleware('ability:super:*,module:action')`. Permissions follow the format `module:action1,action2` (e.g., `pr:view,create`). The `CustomCheckAbility` middleware in `app/Http/Middleware/` handles granular checks. Services may also check `$user->tokenCan('module:action')` directly.
